# 封装师（The Packager）v1.0

你有一段协议文本，想让它被 Agent 读懂。

不是在讲「这是一个很厉害的协议」——而是从「你粘贴进来，四份文件出去」这个具体动作开始。

---

## 一句话描述

把自然语言协议，封装成通用 Agent 平台可读取的 SKILL 格式。输入是协议文本，输出是四份文件。

---

## 如何使用

**第一步：加载协议**

把 `SKILL.md` 全文粘贴给任意支持长文本的 AI。
AI 读完会输出开场白，然后等你粘贴协议。

**第二步：粘贴你的协议**

把任意一份自然语言协议（`.md` / `.txt` / 直接粘贴均可）给 AI。
如协议已在本地文件，告诉 AI 文件路径，它会自行读取。

**第三步：确认类型判定**

AI 会判断协议属于哪类（可参数化 / 不可参数化 / 元协议），输出判定理由。
你说「确认」→ AI 进入封装。
你有异议 → 说「调整」，告诉 AI 应该走哪类。

**第四步：拿到四文件**

AI 一次性输出四份文件全文。你复制保存即可。

```
your-skill/
├── SKILL.md      # Agent 加载的核心文件
├── meta.json     # 机器可读的元数据
├── README.md     # 给人类看的使用说明
└── LICENSE.md    # MIT License（默认）
```

---

## 触发词

- `封装`
- `打包成 SKILL`
- `Package this protocol`
- `封装师`

收到任一触发词，AI 输出开场白，进入第一步。

---

## 适配说明

封装师本身不依赖特定模型——任何能读长文本、能写结构化输出的 AI 都能执行。

但封装出来的 SKILL 文件，适配性取决于原协议的设计：
- 原协议含「禁止 AI 味」「禁止编造」等铁律 → 封装后适配性强
- 原协议无输出约束 → 封装后不同模型执行效果可能有差异

---

## 注意事项

- 封装师不改写协议核心逻辑，只做对外包装
- 原协议未署名的，`meta.json` 中 `author` 字段标注 `[待填写]`，需你手动补充
- 封装完成后，建议先过一遍「班门」审计质量，再对外发布
- 四文件缺一不可——`LICENSE.md` 默认 MIT，原协议指定其他许可证的依原协议

---

> 协议是容器，不是模具。
> 封装师做的事，是给容器找一个合适的盖子。
