# 封装师（The Packager）v2.0

> **启动方式**：在对话框输入 `封装` 或 `打包成 SKILL`，即可启动。

把一段自然语言协议，封装成通用 Agent 平台可读取的 SKILL 格式。
不是改写协议，是给协议一个容器，让 Agent 能读懂它。

---

## 一句话描述

输入任意自然语言协议 → 输出四份标准文件（SKILL.md + meta.json + README.md + LICENSE.md），可直接安装到 Claude Code / WorkBuddy / CNB / OpenClaw / Codex 等平台。

---

## 如何使用

**输入示例**：

```
封装

[粘贴你的协议全文，或告诉封装师文件路径]
```

**输出示例**：

封装师会一次性输出四份文件：

```
your-skill/
├── SKILL.md      # Agent 加载的核心文件
├── meta.json     # 机器可读的元数据
├── README.md     # 给人类看的使用说明
└── LICENSE.md    # MIT License（默认）
```

---

## 完整使用流程

1. **加载封装师**：把 `SKILL.md` 放入对应平台的 skills 目录，或直接将 `封装师_协议_v2.0.md` 全文粘贴给 AI
2. **提供协议**：粘贴协议全文，或告诉封装师文件路径
3. **确认类型**：封装师会判断协议类型（可参数化 / 不可参数化 / 元协议），你确认后进入封装
4. **拿到四文件**：封装师一次性输出四份文件全文，你复制保存即可
5. **（可选）发布前检查**：如计划发布到 GitHub，告诉封装师「发布检查」，它会执行专名清理、路径通用化等检查

---

## 触发词

- `封装`：激活封装师，进入第一步
- `打包成 SKILL`：同上
- `Package this protocol`：英文触发，效果同上
- `确认`：确认协议类型判定，进入封装
- `调整`：修改协议类型判定
- `重新封装`：回到第一步，重新读协议
- `核查`：强制执行封装核查
- `发布检查`：强制执行发布前检查（v2.0 新增）

---

## 适用平台

- ✅ **Claude Code**：放入 `~/.claude/skills/[协议名]/` 或 `.claude/skills/[协议名]/`
- ✅ **WorkBuddy**：放入 `~/.workbuddy/skills/[协议名]/` 或 `.workbuddy/skills/[协议名]/`
- ✅ **CNB**：按 CNB 的 SKILL 格式集成
- ✅ **OpenClaw**：按 OpenClaw 的 SKILL 格式集成
- ✅ **Codex**：将技能目录放入以下任一位置：
  - 仓库级（当前项目）：`$CWD/.agents/skills/[协议名]/`
  - 用户级（所有项目可用）：`$HOME/.agents/skills/[协议名]/`
  - 系统级（机器上所有用户）：`/etc/codex/skills/[协议名]/`
  - Codex 会自动检测技能更改，如未生效，重启 Codex
  - 激活方式：显式调用（输入 `$协议名` 或 `/skills`）或隐式调用（任务与 `description` 匹配时自动选择）
- ⚠️ **不适用**：不支持长文本输入或不支持 SKILL 加载的平台

## 依赖工具

- 无特殊依赖：本协议纯文本协议，无需额外工具

---

## 注意事项

- 封装师**不会改动协议核心逻辑**，只做对外包装
- 封装师**不会评判协议质量**，如需要质量审计，建议先调用协议质量审计员
- 封装后的 SKILL **可以修改**，改完协议正文后，重新跑一遍封装师即可
- 封装后的 SKILL **可以不发布**，私人协议封装后，不上传 GitHub 即可

## 故障排除

**Q：安装后协议不生效？**
A：检查 SKILL.md 是否放在正确的目录下；检查触发词是否与原协议一致；重启 Agent 平台。

**Q：输出结果不符合预期？**
A：检查是否提供了完整的输入；检查是否使用了正确的触发词；检查原协议是否有特殊要求。

**Q：meta.json 解析错误？**
A：检查 meta.json 是否是合法 JSON（可用 JSON 校验工具检查）；检查是否有中文引号或全角字符。

**Q：README.md 里的安装路径不对？**
A：封装师已写明多平台安装路径，如你的平台不在列表中，参考「适用平台」章节手动调整。

**Q：发布到 GitHub 后，别人安装不生效？**
A：检查是否四文件齐全；检查 SKILL.md 是否在正确目录下；检查触发词是否清晰。

## 生态说明

本协议是**通用封装工具**，不依赖特定平台或生态。
如封装的协议本身含有专有名词（如「星云」「墨系列」等），封装师会在 README.md 里增加「生态说明」章节，用 1-2 句话解释。
